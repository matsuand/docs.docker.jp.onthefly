docker trust inspect

読む時間の目安: 10 分

説明

鍵と署名に関する低レベル情報を返します。

利用方法

$ docker trust inspect IMAGE[:TAG] [IMAGE[:TAG]...]

追加説明

docker trust inspectは署名したリポジトリに対して JSON 形式による低レベル情報を提供します。 その情報には署名したイメージタグ、署名者、新たなタグへの署名可能な者の情報などが示されます。

本コマンドの利用例については、以下に示す 利用例の節 を参照してください。

オプション

名前／省略形 デフォルト 説明
--pretty 読みやすい書式で内容を出力します。

利用例

1 つのイメージタグに対する署名についての低レベル詳細情報の取得

docker trust inspectを実行すれば、イメージにおけるトラスト情報を得ることができます。 以下の例ではalpine:latestイメージのトラスﾄ情報を表示します。

        "Name": "Repository",
        "Keys": [

フォーマット設定

inspect による出力にあたっては、デフォルトの JSON 出力ではなく、読みやすいフォーマットにより出力することができます。 これを行うには--prettyオプションを使います。

1 つのイメージタグに対する署名の詳細情報取得

$ docker trust inspect --pretty alpine:latest

SIGNED TAG          DIGEST                                                             SIGNERS
latest              1072e499f3f655a032e88542330cf75b02e7bdf673278f701d7ba61629ee3ebe   (Repo Admin)

Administrative keys for alpine:latest:
Repository Key: 5a46c9aaa82ff150bb7305a2d17d0c521c2d784246807b2dc611f436a69041fd
Root Key:       a2489bcac7a79aa67b19b96c4a3bf0c675ffdf00c6d2fabe1a5df1115e80adce

SIGNED TAG（署名タグ）は署名されているイメージタグのことであり、内容を一意に特定できるDIGESTを持ちます。 SIGNERS（署名者）には署名されているエンティティがすべて表示されます。

上に示される管理鍵（administrative keys）には、トラストのルート鍵（root key）と管理リポジトリ鍵（administrative repository key）が示されます。 この鍵は署名者を変更するため、あるいは署名リポジトリの鍵をローテートするために利用されます。

別のdocker trustコマンドを通じてこのリポジトリに署名者が設定さている場合、docker trust inspect --prettyを実行することで、SIGNER（署名者）として表示され、そのKEYS（鍵）が出力されます。

$ docker trust inspect --pretty my-image:purple

SIGNED TAG          DIGEST                                                              SIGNERS
purple              941d3dba358621ce3c41ef67b47cf80f701ff80cdf46b5cc86587eaebfe45557    alice, bob, carol

上位コマンド

コマンド 説明
docker trust イメージ上の Docker トラストを管理します。

関連コマンド

コマンド 説明
docker trust inspect 鍵と署名に関する低レベル情報を返します。
docker trust key 署名している Docker イメージの鍵を管理します。

docker trust sign イメージに署名します。
docker trust signer Docker イメージに対して署名可能なエンティティを管理します。
